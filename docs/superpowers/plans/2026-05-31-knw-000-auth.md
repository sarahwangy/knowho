# KNW-000 User Auth Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Initialize the Knowho Next.js project from scratch and implement Google OAuth login with session-based auth using NextAuth.js v5 + Prisma + Postgres.

**Architecture:** Next.js 15 App Router project. NextAuth.js v5 (Auth.js) with the Prisma adapter handles OAuth and database sessions — user clicks "用 Google 登录", gets redirected through Google OAuth, lands back as an authenticated session stored in Postgres. All app routes are protected by Next.js middleware; unauthenticated requests redirect to `/login`. First-login hook seeds 8 preset tags for the user.

**Tech Stack:** Next.js 15 · TypeScript · NextAuth.js v5 (`next-auth@beta`) · `@auth/prisma-adapter` · Prisma 5 · PostgreSQL (local dev via `psql` or Docker; production via Vercel Postgres/Neon)

- [ ] **Step 2: Create `auth.ts`**

```typescript
// auth.ts
import NextAuth from "next-auth"
import Google from "next-auth/providers/google"
import { PrismaAdapter } from "@auth/prisma-adapter"
import { prisma } from "@/lib/db"

const PRESET_TAGS = ["读书会", "健身群", "邻居", "工作", "社群", "家人", "朋友", "其他"]

export const { handlers, signIn, signOut, auth } = NextAuth({
  adapter: PrismaAdapter(prisma),
  providers: [Google],
  pages: {
    signIn: "/login",
  },
  callbacks: {
    session({ session, user }) {
      session.user.id = user.id
      return session
    },
    async signIn({ user, account }) {
      if (account?.provider === "google" && user.id) {
        // Seed preset tags for new users (idempotent — upsert won't duplicate)
        await Promise.all(
          PRESET_TAGS.map((name) =>
            prisma.tag.upsert({
              where: { userId_name: { userId: user.id!, name } },
              update: {},
              create: { userId: user.id!, name, isPreset: true },
            })
          )
        )
      }
      return true
    },
  },
})
```

- [ ] **Step 1: Create `middleware.ts`**

```typescript
// middleware.ts
import { auth } from "@/auth"
import { NextResponse } from "next/server"

export default auth((req) => {
  const isLoggedIn = !!req.auth
  const isAuthRoute = req.nextUrl.pathname.startsWith("/login") ||
    req.nextUrl.pathname.startsWith("/api/auth") ||
    req.nextUrl.pathname.startsWith("/s/") // public share pages (KNW-012)

  if (!isLoggedIn && !isAuthRoute) {
    return NextResponse.redirect(new URL("/login", req.url))
  }
  if (isLoggedIn && req.nextUrl.pathname === "/login") {
    return NextResponse.redirect(new URL("/people", req.url))
  }
})

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico|.*\\.png$).*)"],
}
```

## Troubleshooting

**"PrismaClientInitializationError: Can't reach database"**
→ Make sure Postgres is running: `docker ps` or `pg_isready`

**Google OAuth "redirect_uri_mismatch"**
→ In Google Console, confirm the authorized redirect URI is exactly `http://localhost:3000/api/auth/callback/google` (no trailing slash)

**"AUTH_SECRET is not set"**
→ Make sure `.env.local` exists and is in the project root (same level as `package.json`)

**TypeScript error: `Property 'id' does not exist on type 'User'`**
→ Confirm `types/next-auth.d.ts` exists and `tsconfig.json` includes `"types"` in `include`
