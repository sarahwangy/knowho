## 1. 产品愿景

> "Know who you met. Show up like you remembered."

现有 Personal CRM 工具（Clay、Monica、Dex）都面向职场人脉管理，界面复杂、学习成本高。  
Knowho 专注**社区生活圈**——读书会、健身群、邻居、社群——帮普通人在30秒内记录一个新认识的人，并在下次见面前自动提醒你"她上次说她女儿在学钢琴"。

**核心指标（MVP阶段）**

| 指标 | 目标 |
|------|------|
| 新增一个人的时间 | ≤ 30秒 |
| 7日留存率 | ≥ 40% |
| 分享转化（share with a friend） | ≥ 15% |

---

## 2. 用户画像

**主要用户：** 社区活跃者 / 喜欢认识新朋友但容易忘名字的普通人

**痛点：**
- 见面后几天就忘了对方叫什么
- 记不住对方说过的事（生日、孩子、爱好）
- 下次见面尴尬，不知道聊什么

**使用场景：**
- 社群线下活动结束后，地铁上快速录入
- 见面前翻出某人的备注"预热"
- 朋友生日前一天收到提醒

### P0 — 核心层（先做，撑起产品价值）

---

#### 🟠 EPIC-01：快速添加人

---

**[KNW-001] 设计"添加联系人"表单 UI**

- **描述：** 设计并实现最简录入表单，字段：姓名（必填）、认识场合、一句话印象、标签（可多选）。表单需在移动端一屏内完成，不滚动。
- **验收标准：**
  - [ ] 从打开到提交 ≤ 30秒（实测）
  - [ ] 移动端（375px）一屏内可完成
  - [ ] 姓名为唯一必填项，其余可跳过
  - [ ] 提交后跳转到该人的档案页
- **估时：** 3天（含设计稿 + 前端实现）

---

**[KNW-002] 后端 API：创建联系人**

- **描述：** `POST /api/contacts` 接口，接收联系人基础信息，写入数据库，返回新建记录 ID。
- **验收标准：**
  - [ ] 接口返回 201，包含 `contact_id`
  - [ ] 姓名为空时返回 400 + 错误信息
  - [ ] 数据落库，可通过 `GET /api/contacts/:id` 查询
- **估时：** 1天

---

**[KNW-003] 标签系统：预设 + 自定义**

- **描述：** 提供预设标签（读书会、健身群、邻居、工作、社群、家人、其他），支持用户新建自定义标签。标签存为独立实体，联系人与标签多对多关联。
- **验收标准：**
  - [ ] 预设标签在新建表单中可直接点选
  - [ ] 可新建自定义标签并即时使用
  - [ ] 标签列表页可按标签筛选联系人
- **估时：** 2天

---

#### 🟠 EPIC-02：个人档案页

---

**[KNW-004] 联系人档案页 UI**

- **描述：** 展示单个联系人完整信息：头像/姓名、标签、认识场合、重要日期、互动时间线（倒序）、备注。顶部固定"见面前提示卡"入口（P1阶段实现内容，P0先留位置）。
- **验收标准：**
  - [ ] 所有字段正确展示
  - [ ] 时间线按时间倒序排列
  - [ ] 支持编辑入口（跳转编辑页）
  - [ ] 移动端布局正常
- **估时：** 2天

---

**[KNW-005] 后端 API：查询 / 更新 / 删除联系人**

- **描述：** 实现 `GET /api/contacts/:id`、`PATCH /api/contacts/:id`、`DELETE /api/contacts/:id`。
- **验收标准：**
  - [ ] GET 返回完整联系人数据（含标签、日期、时间线）
  - [ ] PATCH 支持部分更新，返回更新后完整记录
  - [ ] DELETE 软删除，不物理清除数据
- **估时：** 1天

---

#### 🟠 EPIC-03：生日 / 重要日期提醒

---

**[KNW-006] 重要日期字段与录入**

- **描述：** 在联系人档案中支持添加多个重要日期，类型：生日、纪念日、自定义。每个日期可设置"提前提醒天数"（默认3天）。
- **验收标准：**
  - [ ] 可添加多个日期
  - [ ] 类型可选：生日 / 纪念日 / 自定义
  - [ ] 提前提醒天数可配置（1/3/7天）
- **估时：** 1.5天

---

**[KNW-007] 提醒推送服务（后端定时任务）**

- **描述：** 每日运行一次定时任务，查询"今日需提醒"的日期记录，通过邮件或推送通知用户。MVP阶段只做邮件推送。
- **验收标准：**
  - [ ] 定时任务每天 09:00 触发
  - [ ] 提醒邮件包含：联系人姓名、日期类型、距离天数
  - [ ] 测试：手动触发定时任务可正常发送邮件
- **估时：** 2天

---

#### 🟠 EPIC-04：互动时间线

---

**[KNW-008] 添加互动记录**

- **描述：** 在联系人档案底部提供快速录入框，记录一次互动：日期（默认今天）+ 一句话/多行文字描述。提交后追加到时间线。
- **验收标准：**
  - [ ] 录入框默认聚焦，减少点击
  - [ ] 日期默认今天，可修改
  - [ ] 提交后时间线实时更新，不刷页
  - [ ] 每条记录可单独删除
- **估时：** 2天

---

**[KNW-009] 后端 API：互动记录 CRUD**

- **描述：** `POST /api/contacts/:id/interactions`、`GET /api/contacts/:id/interactions`、`DELETE /api/interactions/:id`。
- **验收标准：**
  - [ ] 创建返回201含 `interaction_id`
  - [ ] 列表按 `created_at` 倒序返回
  - [ ] 删除返回204
- **估时：** 1天

### P1 — 差异层（别人没做好，你能做）

---

#### 🟡 EPIC-05：见面前提示卡

---

**[KNW-010] 提示卡逻辑与 UI**

- **描述：** 在联系人档案页顶部展示一张"提示卡"，内容自动从时间线最后一条 + 重要日期中抽取，格式："上次你们聊了 ___。Ta 的生日在 ___ 天后。"
- **验收标准：**
  - [ ] 有互动记录时显示最后一条摘要
  - [ ] 有近期日期（30天内）时显示日期提示
  - [ ] 无数据时显示引导语（"还没有记录，见面后写点什么？"）
- **估时：** 1.5天

---

#### 🟡 EPIC-06：语音备注

---

**[KNW-011] 前端语音录入组件**

- **描述：** 在新增联系人表单和互动时间线录入框中，提供麦克风按钮，按住录音，松开自动转文字填入输入框。使用浏览器原生 Web Speech API（或 Whisper API fallback）。
- **验收标准：**
  - [ ] 移动端 Safari / Chrome 均可用
  - [ ] 录音时有视觉反馈（波形或动画）
  - [ ] 转文字结果自动填入文本框，用户可编辑
  - [ ] 失败时有降级提示（"语音识别失败，请手动输入"）
- **估时：** 3天

---

#### 🟡 EPIC-07：好友共享

---

**[KNW-012] 生成共享卡片链接**

- **描述：** 在联系人档案页提供"分享给朋友"按钮，生成一个只读链接，接收方无需注册即可查看该联系人的基础信息（姓名、标签、一句话印象）。敏感字段（私人备注）默认不包含在共享内容中。
- **验收标准：**
  - [ ] 链接有效期7天，过期自动失效
  - [ ] 接收方不登录可查看基础卡片
  - [ ] 用户可手动撤销链接
  - [ ] 不共享私人备注字段
- **估时：** 2天

---

#### 🟡 EPIC-08：社区语言 & 风格规范

---

**[KNW-013] UI 文案审查与替换**

- **描述：** 全站 UI 文案审查，将职场/销售用语替换为社区温度感语言。例：「联系人」→「认识的人」，「跟进」→「下次见面」，「CRM」永远不出现在用户界面。
- **验收标准：**
  - [ ] 所有页面文案无销售/职场术语
  - [ ] 空状态文案温暖友好（非"暂无数据"）
  - [ ] 错误提示友好（非技术错误码）
- **估时：** 1天

### P2 — 进阶层（有资源再做）

---

#### 🔵 EPIC-09：AI 笔记整理

---

**[KNW-014] AI 结构化互动记录**

- **描述：** 用户在时间线中输入一段口语化描述（如"今天见了阿伟，他说他女儿刚考上大学，下个月要去旅行"），AI 自动提取结构化信息并提示："检测到重要日期：下个月旅行。是否添加到提醒？检测到新信息：女儿考上大学。"
- **验收标准：**
  - [ ] 调用 Claude API 分析文本
  - [ ] 提取：人名关联、日期、关键事件
  - [ ] 以卡片形式提示用户确认（不自动写入）
  - [ ] 用户确认后才入库
- **估时：** 4天

---

**[KNW-015] "快冷掉"提醒**

- **描述：** 用户可为每个联系人设置"保持频率"（每周/每月/每季度）。超过设定时间未有互动记录，系统推送提醒："你有 ___ 天没联系 [姓名] 了。"
- **验收标准：**
  - [ ] 频率设置入口在联系人档案页
  - [ ] 推送方式：邮件（MVP）/ 推送通知（后续）
  - [ ] 可一键关闭某人的此功能
- **估时：** 2天

## 5. 技术栈建议

> 基于 Sarah 的全栈背景（React、Golang、Docker、AWS）

| 层 | 建议 | 说明 |
|----|------|------|
| 前端 | React + TypeScript | 已熟悉，移动端用 Tailwind 快速布局 |
| 后端 | Golang + GORM | 已有 dyndns 项目经验，可复用结构 |
| 数据库 | PostgreSQL | 关系型，适合联系人+标签+时间线结构 |
| 推送/邮件 | Resend 或 AWS SES | 简单接入，支持定时任务触发 |
| 语音识别 | Web Speech API（优先）/ OpenAI Whisper（fallback） | 前端直接调用，无后端成本 |
| AI 功能 | Claude API（claude-sonnet-4-20250514） | P2阶段接入 |
| 部署 | Docker + AWS（已熟悉） | 复用现有 DevOps 经验 |
| 认证 | OAuth（Google 登录）| 社区用户门槛最低 |
